# Contributing to Infotainment System

Thank you for considering contributing to this project! Please follow these guidelines to help us maintain a high-quality codebase.

## Getting Started
- Fork the repository and create your feature branch from `main`.
- Run `flutter pub get` to install dependencies.
- Make sure your code builds and passes all tests before submitting a PR.

## Code Style
- Follow Dart and Flutter best practices.
- Use `dart format .` to auto-format your code.
- Use `flutter analyze` to check for linter warnings and errors.
- Add `///` doc comments to all public classes, methods, and properties.
- Use meaningful variable, method, and class names.

## Documentation
- All public classes, methods, and properties must have Dart doc comments (`///`).
- Regenerate API docs with `flutter pub global run dartdoc` if you change public APIs.
- Update the `README.md` and other docs as needed.

## Pull Requests
- Keep PRs focused and small; one feature or fix per PR.
- Reference related issues in your PR description.
- Add tests for new features or bug fixes if possible.
- Ensure your branch is up to date with `main` before submitting.

## Issues
- Search existing issues before opening a new one.
- Provide clear, descriptive titles and detailed information.
- Include screenshots or logs if reporting a bug.

## Community
- Be respectful and constructive in all communications.
- Help others by reviewing PRs and answering questions.

Thank you for helping make this project better!
